# 居宅介護支援事業所向け 総合IDEソフトウェア 仕様草案

## 1. プロジェクト概要

### 1.1 目的
居宅介護支援事業所（ケアマネジャー）の業務効率化を実現する、ブラウザベースの総合業務支援システム（IDE）を開発する。

### 1.2 コンセプト
- **KaigoIDE** - 介護支援専門員のための統合開発環境
- 「ケアプランを"設計"する」という発想で、IDE（統合開発環境）のUXを介護業務に適用

### 1.3 基本要件
| 要件 | 内容 |
|------|------|
| 動作環境 | モダンブラウザ（Chrome, Safari, Edge, Firefox） |
| 対応デバイス | PC、タブレット、スマートフォン（レスポンシブ対応） |
| アーキテクチャ | SaaS型クラウドサービス + PWA対応 |
| オフライン対応 | 一部機能のオフライン動作対応（PWA） |

## 3. モジュール詳細設計

### 3.1 ★【最優先】支援経過記録モジュール（第5表）

#### 概要
介護支援専門員が専門職として残す記録。利用者・家族、サービス担当者、関係機関との関わりを通じて把握・判断した内容を記録。

#### データ構造
```typescript
interface CareRecord {
  id: string;
  clientId: string;                    // 利用者ID
  recordDate: Date;                    // 記録日時
  category: RecordCategory;            // 区分
  content: string;                     // 内容
  relatedOrganization?: string;        // 関係機関
  professionalJudgment?: string;       // 介護支援専門員の判断
  clientFamilyOpinion?: string;        // 利用者・家族の考え
  attachments?: Attachment[];          // 添付ファイル
  createdBy: string;                   // 作成者
  createdAt: Date;
  updatedAt: Date;
}

enum RecordCategory {
  VISIT = '訪問',
  PHONE = '電話',
  FAX = 'FAX',
  MAIL = 'メール',
  CONFERENCE = '会議',
  OTHER = 'その他'
}
```

#### 機能要件
| 機能 | 説明 | 優先度 |
|------|------|--------|
| 記録作成 | 日時・区分・内容の入力 | 高 |
| テンプレート | よく使う文例の登録・呼び出し | 高 |
| 音声入力 | 音声からテキスト変換 | 中 |
| 検索・フィルタ | 日付範囲、区分、キーワード検索 | 高 |
| 時系列表示 | タイムライン形式での閲覧 | 高 |
| PDF出力 | 第5表様式でのPDF出力 | 高 |
| 一括印刷 | 期間指定での一括印刷 | 中 |

#### 画面構成
```

│ 支援経過記録（第5表）                    [+ 新規作成]  │
├─────────────────────────────────────────────────────┤
│ [利用者選択 ▼] [期間: 2024/01 - 2024/12] [検索...]   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ◉ 2024/01/15 10:30  [訪問]                        │
```

### 3.2 ★【最優先】担当者会議モジュール（第4表）

#### 概要
サービス担当者会議の開催記録。会議の要点、検討内容、結論を記録。

#### データ構造
```typescript
interface ConferenceRecord {
  id: string;
  clientId: string;                    // 利用者ID
  conferenceDate: Date;                // 開催日時
  venue: string;                       // 開催場所
  purpose: ConferencePurpose;          // 開催目的

  // 出席者
  attendees: Attendee[];
  absentees?: Absentee[];              // 欠席者（照会対応）

  // 議題・検討内容
  agendaItems: AgendaItem[];

  // 結論
  conclusion: string;
  nextActions?: string;                // 今後の対応
  nextConferenceDate?: Date;           // 次回開催予定

  createdBy: string;
  createdAt: Date;
  updatedAt: Date;
}

interface Attendee {
  name: string;
  organization: string;                // 所属機関
  role: string;                        // 職種
  opinion?: string;                    // 意見
}

interface AgendaItem {
  topic: string;                       // 検討項目
  discussion: string;                  // 検討内容
  decision?: string;                   // 決定事項
}

enum ConferencePurpose {
  NEW_PLAN = '新規ケアプラン作成',
  PLAN_CHANGE = 'ケアプラン変更',
  RENEWAL = '更新認定',
  CATEGORY_CHANGE = '区分変更',
  URGENT = '緊急対応',
  OTHER = 'その他'
}
```

#### 機能要件
| 機能 | 説明 | 優先度 |
|------|------|--------|
| 会議記録作成 | 基本情報・出席者・議題の入力 | 高 |
| 出席者管理 | サービス事業所マスタからの選択 | 高 |
| 照会記録 | 欠席者への事前照会内容記録 | 高 |
| テンプレート | 議題テンプレート | 中 |
| リマインダー | 会議開催前の通知 | 中 |
| PDF出力 | 第4表様式でのPDF出力 | 高 |
| 参加依頼送信 | メール/FAXでの開催通知 | 低 |

#### 画面構成
```
┌─────────────────────────────────────────────────────┐
│ サービス担当者会議の要点（第4表）         [+ 新規作成] │
├─────────────────────────────────────────────────────┤
│ 【基本情報】                                         │
│ 利用者: [山田 太郎 ▼]  開催日: [2024/01/15]          │
│ 開催場所: [利用者自宅        ]                       │
│ 開催目的: [● 新規  ○ 変更  ○ 更新  ○ その他]        │
├─────────────────────────────────────────────────────┤
│ 【出席者】                           [+ 追加]        │
│ ┌──────────────────────────────────────────────┐    │
│ │ ☑ 山田 花子（妻）    - 家族                   │    │

├─────────────────────────────────────────────────────┤
```
