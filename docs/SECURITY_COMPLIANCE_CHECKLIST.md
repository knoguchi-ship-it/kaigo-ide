# セキュリティ準拠チェックリスト（国内リージョン / 準拠優先）

## 前提
- リージョン: 国内（asia-northeast1）
- 監査ログ保持期間: **6ヶ月**
- 監査ログは **Cloud Logging / Cloud Audit Logs** に集約（Sheets 外出しは行わない）

## 1. ガバナンス / 組織
- [ ] 情報管理規程（責任分界、委託管理、SLA）を文書化
- [ ] 事故対応フロー（報告・連絡・封じ込め・再発防止）を文書化
- [ ] BCP / 復旧方針（RTO/RPO）を文書化
- [ ] 個人情報の取扱い（目的外利用禁止、保存期間、削除方針）を文書化

## 2. アクセス制御
- [ ] IAM を最小権限で設計（開発者/運用者/管理者の分離）
- [ ] 管理者操作は監査ログ対象に含める
- [ ] アプリ側で `tenantId` 分離を全 API で強制
- [ ] 重要 API にはロール制御（ADMIN / CARE_MANAGER）を適用

## 3. 認証 / セッション
- [ ] Google OAuth を本番設定に切り替え（開発ログイン無効化）
- [ ] JWT の秘密鍵を Secret Manager に保管
- [ ] トークン有効期限とリフレッシュ運用を定義

## 4. データ保護
- [ ] 通信は TLS 強制（HTTP の遮断）
- [ ] 保存時暗号化（GCP 標準暗号化を前提、必要時 KMS）
- [ ] 機微情報の取り扱い（ログ出力のマスキング）

## 5. 監査・ログ
- [ ] Cloud Audit Logs を有効化（管理操作/設定変更）
- [ ] アプリ監査ログ（認証・主要 CRUD 操作）を出力
- [ ] 監査ログの保持期間を 6ヶ月に設定
- [ ] 監査ログの閲覧権限を最小化
- [ ] 監査ログに ip / userAgent を含める

## 6. 可用性 / 復旧
- [ ] Firestore バックアップ/エクスポートの運用設計
- [ ] 復旧手順（復元・検証）を文書化
- [ ] 障害時の切り戻し手順を定義

## 7. プラットフォーム構成（GCP）
- [ ] Cloud Run を国内リージョンで運用
- [ ] Firestore を国内リージョンで作成
- [ ] Secret Manager を使用（JWT/DB/OAuth）
- [ ] Cloud Logging / Cloud Monitoring を有効化

## 8. アプリ実装（Firestore 移行）
- [ ] Prisma 依存の Repository/Service を Firestore DAO に置換
- [ ] CarePlan/CareRecord/MonitoringRecord のデータ構造を Firestore 向けに整理
- [ ] Firestore インデックスを定義
- [ ] Seed データ投入方法を再設計

## 9. テスト / 監査
- [ ] 監査ログが全主要操作で出力されることを検証
- [ ] 監査ログの保存期間・改ざん防止の確認
- [ ] 権限外アクセスが遮断されることを検証
