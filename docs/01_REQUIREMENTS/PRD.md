## 1. エグゼクティブサマリー

### 1.1 プロダクトビジョン
居宅介護支援専門員（ケアマネジャー）が、いつでもどこでも効率的に業務を遂行できる統合業務支援システムを提供し、介護現場のICT化を推進する。

### 1.2 ミッションステートメント
> 「ケアマネジャーが本来の仕事 ─ 利用者に寄り添う支援 ─ に集中できる環境を創る」

### 1.3 成功指標 (KPIs)

| 指標 | 現状 | 目標 (6ヶ月後) | 目標 (1年後) |
|------|------|----------------|--------------|
| 記録作成時間 | - | 30%削減 | 50%削減 |
| ユーザー満足度 (NPS) | - | +30 | +50 |
| 月間アクティブユーザー | 0 | 100事業所 | 500事業所 |
| 機能利用率（第5表） | - | 80% | 95% |

##### Googleカレンダー連携仕様

**データストレージ方針**: 支援経過記録はGoogleカレンダーをプライマリデータストアとして使用する。

| 項目 | 仕様 |
|------|------|
| 認証方式 | OAuth 2.0（ユーザーごとのGoogleアカウント連携） |
| カレンダー | ユーザーごとに専用カレンダー「KaigoIDE_支援経過記録」を作成 |
| データ形式 | Googleカレンダーイベント + Extended Properties |
| 検索 | Googleカレンダーの検索機能を活用 |
| 閲覧 | KaigoIDE / Googleカレンダーアプリ 両方で閲覧可能 |

**イベントマッピング**:
- タイトル: `[区分] 利用者名様`
- 本文: 記録内容
- 開始日時: 記録日時
- 拡張プロパティ: 利用者ID、区分、関係機関、判断、家族意見

**データストレージ方針**: 担当者会議もGoogleカレンダーをプライマリデータストアとして使用する。

| 項目 | 仕様 |
|------|------|
| カレンダー名 | `KaigoIDE_担当者会議`（支援経過記録とは別カレンダー） |

**ケアプランとの関係**:
- 担当者会議は必ずケアプラン（第1表・第2表）に紐づく
- カレンダー上の配置日 = ケアプラン作成日

#### 3.1.5 ケアプラン簡易管理（Phase 1限定）

**目的**: 担当者会議（第4表）の紐づけに必要な最小限のケアプラン管理機能。
Phase 2でフル機能のケアプラン管理に置き換える。

| ID | 機能 | 説明 | 優先度 |

| FP-004 | ステータス管理 | 下書き/有効/アーカイブの状態管理 | Must |

##### データモデル（Phase 1簡易版）

```typescript
interface CarePlanSimple {
  id: string;                    // プランID（UUID）
  clientId: string;              // 利用者ID
  createdDate: Date;             // ★作成日（担当者会議の配置日に使用）
  version: number;               // 版数（1, 2, 3...）
  purpose: PlanPurpose;          // 新規作成/変更/更新/区分変更
  status: 'draft' | 'active' | 'archived';
  memo?: string;                 // 備考（任意）
  createdAt: Date;
  updatedAt: Date;
}

type PlanPurpose =
  | 'new'           // 新規作成
  | 'change'        // 変更
  | 'renewal'       // 更新
  | 'level_change'; // 区分変更
```

##### 運用フロー

```
1. 利用者詳細画面 → 「新規プラン作成」ボタン
2. 作成日、版数、目的を入力 → 保存
3. 担当者会議作成時に「紐づくプラン」を選択
4. Googleカレンダーにはプラン作成日でイベントが配置される
```

**注意**: Phase 1のケアプラン簡易管理は、第1表・第2表の内容（利用者意向、援助方針、目標等）は含まない。これらはPhase 2で実装する。

## 7. データモデル概要

### 7.1 主要エンティティ

```
┌─────────────┐      ┌─────────────┐
│   Tenant    │──────│    User     │
│  (事業所)    │      │  (ユーザー)  │
└─────────────┘      └─────────────┘
                           │
                           │ 担当
                           ▼
                    ┌─────────────┐
                    │   Client    │
                    │  (利用者)    │
                    └─────────────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                 ▼
  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐
  │  CarePlan   │   │ CareRecord  │   │ Conference  │
  │(ケアプラン)  │   │  (第5表)    │   │  (第4表)    │
  │ ※Phase1簡易 │   └─────────────┘   └─────────────┘
  └─────────────┘                            │
         │                                   │
         │ 1:N 紐づけ                  ┌──────┴──────┐
         │                            ▼             ▼
         └─────────────────────►┌───────────┐ ┌───────────┐
           (plan_id)            │ Attendee  │ │AgendaItem │
                                │ (出席者)   │ │ (議題)    │
                                └───────────┘ └───────────┘
```

**ケアプランと担当者会議の関係**:
- 担当者会議は必ず1つのケアプランに紐づく（plan_id）
- カレンダー上の配置日 = ケアプラン作成日（会議開催日ではない）

### 7.2 主要テーブル

| テーブル | 説明 | 主要カラム |
|----------|------|-----------|
| tenants | 事業所 | id, name, provider_number |
| users | ユーザー | id, tenant_id, email, role |
| clients | 利用者 | id, tenant_id, name, insurer_number, insured_number, care_level |
| care_plans | ケアプラン(簡易) | id, client_id, created_date, version, purpose, status |
| care_records | 支援経過記録 | id, client_id, record_date, category, content |
| conferences | 担当者会議 | id, client_id, **plan_id**, conference_date, venue, purpose |
| attendees | 出席者 | id, conference_id, name, organization |
| agenda_items | 議題 | id, conference_id, topic, discussion |

**注**:
- `care_plans` はPhase 1では簡易版（作成日・版数・目的のみ）
- `conferences.plan_id` により担当者会議はケアプランに紐づく
- 支援経過記録（第5表）・担当者会議（第4表）の実データはGoogleカレンダーに保存
