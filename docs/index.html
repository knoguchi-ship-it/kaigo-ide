Googleカレンダー連携仕様【重要】
データストレージ方針
支援経過記録はGoogleカレンダーをプライマリデータストアとして使用する。
最終的なデータは全てGoogleカレンダー上に残り、Googleカレンダーで検索・閲覧が可能。

連携仕様

項目 仕様
認証方式 OAuth 2.0（ユーザーごとのGoogleアカウント連携）
カレンダー ユーザーごとに専用カレンダーKaigoIDE_支援経過記録を自動作成
データ形式 Googleカレンダーイベント + Extended Properties
検索機能 Googleカレンダーの検索機能を活用
閲覧方法 KaigoIDE / Googleカレンダーアプリ 両方で閲覧可能

// Googleカレンダーイベントへのマッピング
interface CareRecordEvent {
  // === 標準フィールド ===
  summary: string;              // "[訪問] 山田太郎様"
  description: string;          // 記録内容（本文）
  start: {
    dateTime: string;           // 記録日時
    timeZone: 'Asia/Tokyo';
  };
  end: {
    dateTime: string;           // 記録日時 + 30分
    timeZone: 'Asia/Tokyo';
  };

  // === 拡張プロパティ（カスタムデータ）===
  extendedProperties: {
    private: {
      kaigoide_type: 'care_record';    // レコード識別子
      kaigoide_version: '1';           // データバージョン
      client_id: string;               // 利用者ID
      client_name: string;             // 利用者名
      category: RecordCategory;        // 区分
      related_org?: string;            // 関係機関
      judgment?: string;               // 判断
      family_opinion?: string;         // 家族意見
    }
  }
}

enum RecordCategory {
  VISIT = '訪問',
  PHONE = '電話',
  FAX = 'FAX',
  MAIL = 'メール',
  CONFERENCE = '会議',
  OTHER = 'その他'
}

★ 担当者会議記録（第4表）

PRD - 最優先機能

概要
サービス担当者会議の開催記録。会議の要点、検討内容、結論を記録する。
Googleカレンダー連携対象 - ケアプラン（第1表・第2表）の作成日に紐づけて保存。
Googleカレンダー連携仕様【重要】
日付の紐づけ
担当者会議のカレンダー上の日付は「紐づくケアプラン（第1表・第2表）の作成日」とする。
これにより、ケアプラン作成の履歴がカレンダー上で可視化される。

カレンダー名 KaigoIDE_担当者会議（支援経過記録とは別カレンダー）
イベント日付 ケアプラン作成日（終日イベント）
タイトル形式 [開催目的] 利用者名様 担当者会議
本文 結論 + 出席者・議題（JSON形式）
PDF出力 厚労省第4表様式でダウンロード可能

// Googleカレンダーイベントへのマッピング
interface ConferenceEvent {
  // === 標準フィールド ===
  summary: string;              // "[新規] 山田太郎様 担当者会議"
  description: string;          // 結論 + JSON(出席者・議題)
  start: { date: string };      // ★ケアプラン作成日（終日イベント）
  end: { date: string };        // ★ケアプラン作成日

  // === 拡張プロパティ ===
  extendedProperties: {
    private: {
      kaigoide_type: 'conference';
      kaigoide_version: '1';
      client_id: string;               // 利用者ID
      client_name: string;             // 利用者名
      plan_id: string;                 // ★紐づくケアプランID
      plan_created_date: string;       // ★ケアプラン作成日
      conference_date: string;         // 実際の会議開催日
      purpose: ConferencePurpose;      // 開催目的
      venue: string;                   // 開催場所
      conclusion: string;              // 結論（1024文字まで）
    }
  }
}

// 本文(description)に格納するJSON
interface ConferenceDescription {
  attendees: Attendee[];        // 出席者一覧
  absentees: Absentee[];        // 欠席者（照会）
  agendaItems: AgendaItem[];    // 検討項目
  nextActions?: string;         // 今後の対応
}

enum ConferencePurpose {
  NEW_PLAN = '新規ケアプラン作成',
  PLAN_CHANGE = 'ケアプラン変更',
  RENEWAL = '更新認定',
  CATEGORY_CHANGE = '区分変更',
  URGENT = '緊急対応',
  OTHER = 'その他'
}

重要な技術決定とその理由

ID タイトル 決定内容 ステータス

ADR-0001: React + TypeScript

決定理由

  • 最大のエコシステムとコミュニティ
  • TypeScriptによる型安全性
  • PWA対応が容易
  • 採用市場が最も大きく人材確保しやすい

検討した選択肢

  • Vue.js + TypeScript
  • Next.js
  • Angular
ADR-0003: 状態管理

決定理由

  • TanStack Query: サーバー状態に特化、キャッシュ自動化
  • Zustand: 学習コスト低、ボイラープレート少
  • 各ライブラリが得意な領域に特化

検討した選択肢

  • Redux + RTK Query
  • SWR
  • Jotai
ADR-0004: NestJS

決定理由

  • モジュール構造がKaigoIDEのモジュール分割方針と合致
  • TypeScriptファースト
  • DIコンテナによる高いテスタビリティ
  • OpenAPI自動生成

検討した選択肢

  • Express.js
  • Fastify
  • Hono
ADR-0006: JWT認証

決定理由

  • SPA/PWAでの認証に適している
  • ステートレスでスケーラビリティが高い
  • オフライン対応との整合性

セキュリティ対策

  • Access Token: 15分有効、メモリ保持
  • Refresh Token: 7日有効、HTTP Only Cookie
  • トークンローテーション
ADR-0007: Googleカレンダー連携（第5表・第4表）【重要】

決定内容

Googleカレンダーを支援経過記録（第5表）・担当者会議（第4表）のプライマリデータストアとして採用

カレンダー構成

カレンダー 対象 イベント日付
KaigoIDE_支援経過記録 第5表 記録日時（実際の日時）
KaigoIDE_担当者会議 第4表 ケアプラン作成日

決定理由

  • Googleカレンダーで全ての記録が検索・閲覧可能
  • 担当者会議をケアプラン作成日に配置→計画作成の履歴が可視化
  • 厚労省様式PDF出力でデータ復元可能
  • ユーザー自身のGoogleアカウントにデータが残る

実装方式

  • OAuth 2.0でユーザーのGoogleアカウントと連携
  • ユーザーごとに2つの専用カレンダーを作成
  • Extended Propertiesでカスタムデータを保持
  • 複雑なデータ（出席者・議題）は本文にJSON格納
