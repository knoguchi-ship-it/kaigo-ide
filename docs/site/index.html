Documentation Portal

文書を迷わず読むための共通入口

KaigoIDE の運用ルール、現在地、文字コード方針を、Markdown の正本と視覚ページの両方から理解できるように整理した静的ドキュメントサイトです。

正本は Markdown HTML は視覚導線 GitHub Pages 公開中 UTF-8 前提

最重要ルール

着手前に最新情報を Web で確認し、根拠付き提案を出し、承認後に docs-first で進めます。

  • Web 調査必須
  • 提案必須
  • 承認前実装禁止

現在地

現行 API は Firestore ベースで、AI は `dev=Gemini Developer API`、`prod target=Vertex AI` の方針です。

  • 支援経過記録 API 実装済み
  • モニタリング入力 API 実装済み
  • 担当者会議は未実装
  • ドキュメントサイトは GitHub Pages で公開中

今の重点

文字化けは「表示問題」と「実ファイル破損」を分けて扱い、再発防止の設定を固定します。

  • UTF-8 を標準化
  • `.editorconfig` 導入
  • `.gitattributes` 導入
