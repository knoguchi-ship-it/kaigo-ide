GCP 主軸の運用前提を揃える

このページは、個人開発から本番運用までの AI / 認証 / Firestore / 環境変数の前提を短く把握するための要約です。

結論

個人開発では `Gemini Developer API`、本番や実データでは `Vertex AI` を使います。production では `DEV_LOGIN_ENABLED=false` を維持します。

開発 / 個人運用

  • `AI_PROVIDER=gemini`
  • 匿名化データのみを前提
  • Firestore + local dev

本番 / 実データ

  • `AI_PROVIDER=vertex`
  • `DEV_LOGIN_ENABLED=false`
  • GCP 認証と環境変数バリデーション必須

本番前チェック

  1. Firestore と Vertex AI の GCP プロジェクト整合を確認する
  2. service account か metadata 認証を固定する
  3. 開発ログインが無効になっていることを確認する
  4. Cloud Run 前提の環境変数を確認する
