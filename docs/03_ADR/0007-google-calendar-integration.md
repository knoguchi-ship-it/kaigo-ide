# ADR-0007: Googleカレンダー連携（第5表・第4表）

**ステータス**: Accepted
**日付**: 2026-01-29
**最終更新**: 2026-01-29
**決定者**: [アーキテクト名]

## コンテキスト

支援経過記録（第5表）および担当者会議（第4表）のデータ保存方式を検討する必要がある。以下の要件が示された：

1. Googleカレンダーと連携する
2. ログインユーザーごとの特定のカレンダーでデータを持つ
3. 最終的なデータはすべてGoogleカレンダー上に残る
4. Googleカレンダーで全ての記録が検索・閲覧できる
5. 担当者会議（第4表）もGoogleカレンダー連携する
6. 担当者会議は紐づくケアプラン（第1表・第2表）の作成日に配置する
7. 担当者会議は厚労省様式でPDFダウンロード可能とする

## 決定

**Googleカレンダーを支援経過記録（第5表）・担当者会議（第4表）のプライマリデータストアとして採用する。**

## カレンダー構成

| カレンダー名 | 用途 | イベント日付 |
|-------------|------|-------------|
| KaigoIDE_支援経過記録 | 第5表の記録 | 記録日時（実際の日時） |

## データ構造

### 支援経過記録（第5表）イベント

```typescript
interface CareRecordEvent {
  // === 標準フィールド ===
  summary: string;              // "[訪問] 山田太郎様"
  description: string;          // 記録内容
  start: { dateTime: string };  // 記録日時
  end: { dateTime: string };    // 記録日時 + 30分

  // === 拡張プロパティ ===
  extendedProperties: {
    private: {
      kaigoide_type: 'care_record';
      kaigoide_version: '1';
      client_id: string;
      client_name: string;
      category: RecordCategory;
      related_org?: string;
      judgment?: string;
      family_opinion?: string;
    }
  }
}
```

### 担当者会議（第4表）イベント

```typescript
interface ConferenceEvent {
  // === 標準フィールド ===
  summary: string;              // "[新規] 山田太郎様 担当者会議"
  description: string;          // 会議概要（結論等）
  start: { date: string };      // ケアプラン作成日（終日イベント）
  end: { date: string };        // ケアプラン作成日

  // === 拡張プロパティ ===
  extendedProperties: {
    private: {
      kaigoide_type: 'conference';
      kaigoide_version: '1';
      client_id: string;
      client_name: string;
      plan_id: string;                    // 紐づくケアプランID
      plan_created_date: string;          // ケアプラン作成日
      conference_date: string;            // 実際の会議開催日
      purpose: ConferencePurpose;         // 開催目的
      venue: string;                      // 開催場所
      conclusion: string;                 // 結論
      // 出席者・議題は本文(description)にJSON形式で格納
    }
  }
}

// 本文(description)の構造
interface ConferenceDescription {
  attendees: Attendee[];        // 出席者一覧
  absentees: Absentee[];        // 欠席者（照会）
  agendaItems: AgendaItem[];    // 検討項目
  nextActions?: string;         // 今後の対応
}
```

interface CarePlan {

## 担当者会議のPDF出力

担当者会議（第4表）は、Googleカレンダーからデータを取得し、**厚労省標準様式のPDF**としてダウンロード可能。

### PDF出力フロー

```
1. KaigoIDEで担当者会議を選択
2. 「PDF出力」ボタンをクリック
3. Googleカレンダーからイベントデータを取得
4. 拡張プロパティ + description(JSON) からデータを復元
5. 厚労省第4表様式でPDFを生成
6. ダウンロード
```

## 選択肢

### 選択肢1: Googleカレンダーをプライマリストレージ（採用）
- **概要**: 第5表・第4表をGoogleカレンダーイベントとして保存
- **メリット**:
  - Googleカレンダーで全記録を検索・閲覧可能
  - モバイルアプリでも確認可能
  - ユーザーがデータを所有
- **デメリット**:
  - API制限
  - 複雑なデータは本文にJSON格納が必要

### 選択肢2: PostgreSQL + 同期
- 同期の整合性問題があるため不採用

## 決定理由

1. **要件適合**: Googleカレンダーで検索・閲覧の要件を満たす
2. **ケアプラン連携**: 担当者会議をケアプラン作成日に配置することで、計画作成の流れが可視化される
3. **PDF出力**: データを復元して厚労省様式で出力可能

## 影響

### ポジティブな影響
- ケアプラン作成日に担当者会議が表示され、計画の履歴が分かりやすい
- Googleカレンダーで「この日に何のプランを作ったか」が一目瞭然
- 標準様式PDFで印刷・提出可能

### ネガティブな影響
- 複雑なデータ（出席者・議題）はJSON格納となり、直接編集が難しい
- ケアプラン（第1表・第2表）の管理が必要（Phase 2で実装予定だがIDは先行して必要）

### リスクと軽減策

| リスク | 軽減策 |
|--------|--------|
| ケアプランIDの管理 | Phase 1ではシンプルなID採番で対応 |
| JSON解析エラー | バージョン管理、スキーマ検証 |
| PDF様式変更 | テンプレート分離、更新対応 |
