# ADR-0005: データベースの選定

**ステータス**: Accepted
**日付**: 2026-01-29
**決定者**: [アーキテクト名]

## コンテキスト

KaigoIDEのデータ永続化層のデータベースを選定する必要がある。

要件：
- 介護記録、利用者情報等の構造化データの保存
- マルチテナント（事業所単位の分離）のサポート
- 高い信頼性と耐久性
- 将来的なスケーリング
- 日本語全文検索のサポート（検索機能）
- JSON型データの柔軟な格納（設定、メタデータ等）

## 決定

**PostgreSQL をメインデータベースとして採用する。**

ORMには **Prisma** を使用する。

## 選択肢

### 選択肢1: PostgreSQL
- **概要**: オープンソースのオブジェクトリレーショナルデータベース
- **メリット**:
  - 高い信頼性と実績
  - JSON/JSONB型のネイティブサポート
  - 日本語全文検索（pg_trgm、pgroonga）
  - 拡張機能が豊富
  - AWS RDS、Cloud SQL等のマネージドサービス
  - Prismaの最もサポートが良いDB
- **デメリット**:
  - MySQLより設定が複雑
  - 水平スケーリングが難しい

### 選択肢2: MySQL
- **概要**: 最も普及しているRDBMS
- **メリット**:
  - 広く使われており情報が多い
  - 高速な読み取り性能
  - シンプルな運用
- **デメリット**:
  - JSON型のサポートがPostgreSQLより弱い
  - 全文検索がPostgreSQLより弱い

### 選択肢3: MongoDB
- **概要**: ドキュメント指向NoSQLデータベース
- **メリット**:
  - スキーマレスで柔軟
  - 水平スケーリングが容易
  - JSONネイティブ
- **デメリット**:
  - トランザクションの制限
  - リレーションの表現が難しい
  - 介護記録のようなリレーショナルデータに不向き

### 選択肢4: SQLite
- **概要**: ファイルベースの軽量データベース
- **メリット**:
  - セットアップ不要
  - 軽量
  - 開発環境に最適
- **デメリット**:
  - 同時接続に制限
  - 本番環境には不向き

## 決定理由

1. **データモデルの適合性**: 利用者、記録、会議等のリレーショナルデータを扱うため、RDBMSが最適
2. **JSON対応**: 設定データや柔軟なメタデータの格納にJSONB型が便利
3. **全文検索**: pg_trgmやpgroongaで日本語の全文検索が可能
4. **Prismaとの親和性**: PrismaはPostgreSQLを最もよくサポート
5. **マネージドサービス**: 主要クラウドでマネージドサービスが利用可能
6. **実績**: 介護・医療システムでの採用実績が多い

## ORM選定: Prisma

### Prismaの採用理由
1. **型安全性**: スキーマから TypeScript の型が自動生成
2. **直感的なAPI**: クエリビルダーが使いやすい
3. **マイグレーション**: スキーマベースのマイグレーション管理
4. **Prisma Studio**: GUIでデータ確認が可能
5. **NestJSとの統合**: `@prisma/client` を簡単に注入可能

## 影響

### ポジティブな影響
- 複雑なリレーションを適切にモデリング可能
- Prismaにより型安全なデータアクセス
- マイグレーションの管理が容易
- 将来的な全文検索の拡張が可能

### ネガティブな影響
- PostgreSQLの運用知識が必要
- Prismaへの依存（ロックイン）
- 水平スケーリングには追加の工夫が必要

### リスクと軽減策
| リスク | 軽減策 |
|--------|--------|
| 運用の複雑さ | マネージドサービス（RDS等）の利用 |
| Prismaの制限 | 複雑なクエリは$queryRaw使用 |
| パフォーマンス | インデックス設計、クエリ最適化 |
